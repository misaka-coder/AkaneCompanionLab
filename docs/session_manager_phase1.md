# 对话系统（一期）

## 目标

把危险的“新会话”技术开关升级成用户能理解、能切回旧对话的产品能力。

一期只解决三件事：

1. 新建对话
2. 切换旧对话
3. 重命名当前对话

## 产品边界

这一期明确遵守以下约束：

- 前端不展示 `session_id`、`profile_user_id`、UUID 等技术术语
- 对话列表只显示标题
- 不显示摘要预览
- 不显示副标题、时间信息、技术信息
- 不在切换对话时自动触发总结、压缩或任何额外记忆逻辑
- 切换对话仅表示切换聊天线程，不改变既有记忆机制

## 数据结构

新增 `chat_sessions` 表：

- `session_id`
- `profile_user_id`
- `display_title`
- `created_at`
- `updated_at`

## 后端接口

### `POST /sessions/ensure`

用途：

- 首次打开当前对话时确保其存在
- 新建对话后创建空白对话
- 切回旧对话时读取其状态

返回：

- `session`
- `sessions`
- `messages`
- `latest_final_json`

### `POST /sessions/rename`

用途：

- 重命名当前对话

返回：

- `session`
- `sessions`

### `GET /sessions`

用途：

- 获取当前用户的对话列表

## 前端行为

设置面板中的“对话”区只展示：

- `新建对话`
- `重命名`
- 标题列表

当前对话使用高亮样式表示。

切换对话时：

1. 更新本地保存的 `session_id`
2. 请求 `/sessions/ensure`
3. 恢复该对话最近一段对白
4. 恢复该对话最近一次舞台状态（若存在）

## 当前已知限制

- 目前恢复的是最近一段消息窗口，不是无限长完整记录
- 若某段旧对话没有可用的 `latest_final_json`，会回退到默认舞台欢迎语
- `history` 面板仍显示当前对话的对白回看，不是“对话列表”；对话列表在设置面板内
