# 资源协议说明

这份文档讲“资源怎么放、怎么命名、程序怎么扫”。

## 1. 扫描范围

当前运行时主要扫描：

- `web/assets/scenes`
- `web/assets/characters`
- `web/assets/bgm`
- `web/assets/backgrounds`

说明：

- `scenes` 是推荐的新结构。
- `backgrounds` 是旧结构兼容层。

## 2. 场景目录规范

推荐结构：

```text
web/assets/scenes/
  school/
    meta.json
    classroom/
      meta.json
      morning.png
      evening.png
    rooftop/
      meta.json
      sunset.png
      night.png
```

含义：

- 一级目录：大场景 `major`
- 二级目录：子场景 `minor`
- 图片文件：背景变体 `background`

## 3. 角色目录规范

推荐结构：

```text
web/assets/characters/
  校服/
    meta.json
    normal.png
    shy.png
    smug.png
  私服/
    meta.json
    normal.png
    cry.png
```

含义：

- 一级目录：服装 `outfit`
- 图片文件：同一服装下的表情 `emotion`

硬规则：

- `emotion` 只负责同服装差分
- 不允许用 `emotion` 代替换衣服

## 4. BGM 目录规范

推荐结构：

```text
web/assets/bgm/
  school/
    classroom/
      morning.ogg
      evening.ogg
    rooftop/
      sunset.ogg
```

原则：

- 尽量和 `scenes` 层级保持一致
- 同一 `minor` 下的 BGM 尽量和背景变体对应

## 5. meta.json 放置位置

目录级 meta：

- `web/assets/scenes/<major>/meta.json`
- `web/assets/scenes/<major>/<minor>/meta.json`
- `web/assets/characters/<outfit>/meta.json`

单文件 sidecar meta：

- `web/assets/scenes/<major>/<minor>/<background>.meta.json`
- `web/assets/bgm/<major>/<minor>/<track>.meta.json`
- `web/assets/characters/<outfit>/<emotion>.meta.json`

## 6. 兼容规则

如果旧资源还是这种命名：

```text
web/assets/scenes/home/卧室_白天.png
web/assets/scenes/home/卧室_夜晚.png
```

程序会自动归并成：

- `major = home`
- `minor = 卧室`
- `background = morning / night`

## 7. 说明书和配置的边界

- 写给人看的说明书：放 `documents/`
- 给程序读的配置：放 `web/assets/**/meta.json`

不要把这两类混在一起。
